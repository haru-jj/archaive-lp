多くの製造業が、図面をファイルサーバーの共有フォルダで管理しています。最初はそれで十分です。問題は、それが「いつか破綻する」運用だということ。破綻のサインを知っておくと、手遅れになる前に手を打てます。

## フォルダ運用が破綻する理由

フォルダは、図面が増えるほど階層が深くなります。「製品別」「年度別」「顧客別」と分け始めると、同じ図面をどこに置くべきか迷い、人によって置き場所がばらつきます。結果、目的の図面にたどり着けず、似たファイル名で版を取り違え、同じ図面が複数の場所に重複していきます。

## 破綻のサイン

次のような兆候が出てきたら、フォルダ運用の限界が近いと考えられます。

- 図面を探すのに時間がかかるようになった
- 最新版がどれか分からず、版を間違えることがある
- 担当者しか目的の図面の場所を知らない
- 同じような図面があちこちに重複している

これらは、検索性と版管理という、フォルダ運用が構造的に持てない機能の欠如から来ています。

## 破綻したフォルダ運用の移り先

ファイルサーバーのフォルダ運用は、図面が増え人が増えるほど、命名のばらつきと階層の深さで探せなくなります。この限界の先で、図面の中身から探せる管理を用意するのがARCHAIVEです。フォルダ構造を作り込む運用から、検索に頼る運用へ移す選択肢にあたります。
