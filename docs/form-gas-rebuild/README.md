# フォーム受信 GAS 再構築一式

LP の **資料請求 (`/download`)** と **無料トライアル申し込み (`/apply`)** の送信が、
受信側 GAS / スプレッドシート（退職者の個人アカウント所有）の消滅で停止した。
これを**会社共有アカウント**で 0 から作り直すための資料一式。

旧スクリプトは参考に留め、**現行 LP コードが実際に送る内容を正**として再設計している。

## 中身

| ファイル | 用途 |
|---------|------|
| `HANDOFF.md` | **まずこれ。** 作成〜デプロイ〜env差し替え〜疎通確認の手順書（チェックリスト・トラブルシュート付き） |
| `SPEC.md` | データ仕様。各フォームの送信フィールド、GAS が返すべきレスポンス契約、シート列定義 |
| `gas/Code.gs` | 新 GAS 本体（download/apply 受信 → シート追記 → 通知メール →（任意）自動返信）。Apps Script にそのまま貼る |
| `gas/appsscript.json` | GAS マニフェスト（タイムゾーン・権限スコープ・Web アプリ公開設定）。任意 |
| `sheets/資料請求_header.csv` | 資料請求シートのヘッダー行（手で用意する場合用。GAS が自動生成もする） |
| `sheets/無料トライアル_header.csv` | 無料トライアルシートのヘッダー行 |

## 30秒サマリ

1. 会社共有アカウントでスプレッドシート作成（タブ名 `資料請求` / `無料トライアル`）
2. Apps Script に `gas/Code.gs` を貼る
3. スクリプトプロパティ `NOTIFICATION_EMAIL`（通知先）を設定
4. ウェブアプリとしてデプロイ（**アクセス = 全員** が必須）
5. 発行 URL を LP の env `GAS_WEB_APP_URL`（`.env.local` + 本番）に差し替えて再デプロイ
6. `/download` `/apply` から実送信して、シート追記＋通知メールを確認

詳細は `HANDOFF.md`。

## 対象スコープの注意

- 動いているフォームは **資料請求 / 無料トライアルの2つだけ**。
- 「お問い合わせ」(`useContactForm` / `/api/contact`) は**どのページからも呼ばれていない死にコード**のため対象外。
  復活させる場合の方針は `HANDOFF.md` 末尾「将来の課題」に記載。
